muvid.events

Surface fal progress events into the muvid project.

Whenever muvid invokes fal (during render, render_environment, transcribe for transcripts that arrive via fal, etc.), the falaw.events stream is written to .muvid/fal_events.jsonl so:

  • muvid status can show “currently running …” and recent per-shot timings;

  • the local UI can stream the same events over SSE without setting up its own subscriber;

  • post-mortem of long renders is just cat .muvid/fal_events.jsonl.

The integration is opt-in via log_fal_events_to(), which yields a context manager that subscribes / unsubscribes around the with block. muvid.facade.render wraps every render call in one.
